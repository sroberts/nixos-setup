# hosts/

One directory per machine. Each holds everything that is specific to *that*
physical computer; `configuration.nix` and `home.nix` (in the repo root) are
host-agnostic and shared by every host here.

```
hosts/
  sjr-fw13/
    default.nix                 # hostname, nixos-hardware model module, swap/resume
    hardware-configuration.nix  # generated by nixos-generate-config (committed)
```

`flake.nix` **auto-discovers** these: every directory under `hosts/` becomes
`nixosConfigurations.<dirname>`. Adding a machine means adding a directory —
you never edit `flake.nix`.

## Why hardware-configuration.nix is committed

Nix flakes only evaluate files that Git tracks, so a per-host
`hardware-configuration.nix` has to be committed for `nixos-rebuild --flake`
to see it. The file contains LUKS **UUIDs** and the filesystem layout — those
are device *identifiers*, not secrets (a UUID gives no one access to an
encrypted volume; the passphrase does). Committing them makes the repo a true
source of truth: the running machine is fully described by these files.

## What goes in `default.nix` vs the shared files

| Belongs in `hosts/<name>/` | Belongs in `configuration.nix` / `home.nix` |
|---|---|
| `hardware-configuration.nix` (disk UUIDs, filesystems, swapDevices) | Bootloader, services, system packages |
| `networking.hostName` | The `sroberts` user, PipeWire, niri/Noctalia |
| The `nixos-hardware` module for the exact model | Everything else that's identical on every host |
| Hibernation swap LUKS unlock + `boot.resumeDevice` | |

## Standing up a new machine

1. **Install a base NixOS** and boot it (see `../INSTALL.md` for the Framework
   13 Calamares path — encrypted disk, "Swap with Hibernate", user `sroberts`).
2. **Clone this repo and make it the live config:**
   ```bash
   nix-env -iA nixos.git
   git clone https://github.com/sroberts/nixos-setup.git ~/nixos-setup
   sudo rm -rf /etc/nixos && sudo ln -s ~/nixos-setup /etc/nixos
   ```
3. **Scaffold the host directory** (copies this machine's generated hardware
   config in, writes a `default.nix`, auto-detects the swap LUKS UUID when it
   can, and git-adds both):
   ```bash
   cd ~/nixos-setup
   scripts/new-host.sh            # uses the current hostname
   # or: scripts/new-host.sh my-other-laptop
   ```
4. **Review `hosts/<hostname>/default.nix`:**
   - Set the right [`nixos-hardware`](https://github.com/NixOS/nixos-hardware)
     module for the model (the template defaults to the Framework 13 AMD
     Ryzen 7040 — `framework-amd-ai-300-series` for a Ryzen AI 300, etc.).
   - Confirm or complete the hibernation swap block. If `new-host.sh` found
     active swap on a `luks-<uuid>` mapper it's filled in; otherwise find the
     swap partition's LUKS UUID and fill it in:
     ```bash
     swapon --show
     lsblk -o NAME,TYPE,FSTYPE,UUID
     ```
     (Drop the block entirely if you don't want hibernation — suspend-to-RAM
     still works.)
5. **Build it:**
   ```bash
   sudo nixos-rebuild switch --flake .#<hostname>
   ```
6. **Commit** the new host directory (and `flake.lock` if the build generated
   one):
   ```bash
   git add hosts/<hostname> flake.lock
   git commit -m "hosts: add <hostname>"
   git push
   ```

## Determinism note

Because every host shares `configuration.nix` + `home.nix` and pins the same
`flake.lock`, two machines built from the same commit get the *same* package
set, services, and dotfiles. They differ only by the contents of their
`hosts/<name>/` directory — which is exactly the hardware that genuinely
differs between them.
